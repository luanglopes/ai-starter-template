# AGENTS.md

## Scope

Rules for agents working inside the documentation folder.

## Documentation Rules

- Write docs for humans first: clear, direct, and task-oriented.
- Keep documentation factual and synchronized with implemented behavior.
- Prefer updating existing docs over creating duplicate pages.
- Use consistent terminology across files.
- Mark assumptions or TBD items explicitly instead of presenting them as facts.

## Change Discipline

- Do not document features that are not implemented unless the user asks for a proposal/spec draft.
- When behavior changes, update related docs in the same task when feasible.
- Avoid copying large code blocks when a concise example is enough.
- Keep examples runnable or obviously illustrative.

## Structure and Style

- Use short sections and skimmable headings.
- Prefer checklists and examples for operational steps.
- Avoid repository governance or agent-policy content here unless specifically requested.
